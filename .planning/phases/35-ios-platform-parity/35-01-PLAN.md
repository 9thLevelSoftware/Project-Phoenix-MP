<objective>
Fix two BLOCKER iOS database issues: the schema version mismatch that wipes the database every launch (B7), and the missing formScore column that crashes session queries (B8).
</objective>

<context>
## B7: CURRENT_SCHEMA_VERSION = 21 vs SQLDelight version = 22
- **File:** `shared/src/iosMain/kotlin/com/devil/phoenixproject/data/local/DriverFactory.ios.kt` (line 57)
- `CURRENT_SCHEMA_VERSION = 21L` but SQLDelight `version = 22` in build.gradle.kts
- iOS DriverFactory health check compares `user_version < CURRENT_SCHEMA_VERSION`
- Since live schema is 22, the check always triggers a purge-and-recreate
- **Fix:** Update `CURRENT_SCHEMA_VERSION = 22L`

## B8: formScore column missing from iOS manual schema
- **File:** Same `DriverFactory.ios.kt`, in `createAllTables` / `ensureAllColumnsExist` sections
- `VitruvianDatabase.sq` declares `formScore INTEGER` on `WorkoutSession`
- Multiple queries project `formScore` — crash with "no such column" on iOS
- **Fix:** Add `safeAddColumn(driver, "WorkoutSession", "formScore", "INTEGER")` to `ensureAllColumnsExist`
- Also add `formScore INTEGER` to the `WorkoutSession` CREATE TABLE in `createAllTables`
- Check for any other columns added in recent migrations that might be missing from the iOS manual schema
</context>
