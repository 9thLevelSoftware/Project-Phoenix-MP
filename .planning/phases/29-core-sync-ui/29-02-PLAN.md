<objective>
Add Ktor HTTP client keep rules to ProGuard/R8 configuration to prevent release builds from stripping the sync layer's HTTP engine. Without these rules, the app crashes at runtime with "No suitable HttpClient engine found" when attempting cloud sync.
</objective>

<context>
- Phase context: `.planning/phases/29-core-sync-ui/29-CONTEXT.md`
- Implementation plan: `docs/plans/mvp-cloud-sync-mobile.md` (Section 4)
- Branch: `MVP`
- The sync layer uses Ktor 3.3.3 with OkHttp engine (Android) and Darwin engine (iOS)
- `androidApp/proguard-rules.pro` has comprehensive rules for Koin, Serialization, Coroutines, SQLDelight, Kable BLE, Multiplatform Settings, Compose, MediaPipe, CameraX — but NOT Ktor
- This issue only manifests in release builds (debug builds don't use ProGuard/R8)
- Ktor uses service loading and reflection for engine discovery, which R8 strips by default
</context>

<tasks>
### Task 1: Add Ktor ProGuard Keep Rules

**File**: `androidApp/proguard-rules.pro`
**Action**: Add the following Ktor keep rules. Place them after the existing Koin/Serialization/Coroutines section and before the SQLDelight section (or at the end of the file):

```proguard
# Ktor HTTP Client (sync layer)
-keep class io.ktor.** { *; }
-keep class io.ktor.client.** { *; }
-keep class io.ktor.client.engine.** { *; }
-keep class io.ktor.client.plugins.** { *; }
-keep class io.ktor.serialization.** { *; }
-keep class io.ktor.utils.** { *; }
-dontwarn io.ktor.**
```

**Verify**: The rules are syntactically correct (no ProGuard warnings during build).

**Done when**: Ktor keep rules are present in proguard-rules.pro.

---

### Task 2: Build Release APK and Verify No Ktor Stripping

**Action**: Build a release APK using debug signing (for local testing) and verify Ktor classes survive R8 minification.

```bash
./gradlew :androidApp:assembleRelease \
    -Pandroid.injected.signing.store.file=$HOME/.android/debug.keystore \
    -Pandroid.injected.signing.store.password=android \
    -Pandroid.injected.signing.key.alias=androiddebugkey \
    -Pandroid.injected.signing.key.password=android
```

**Verify**: Build succeeds. The APK size should be slightly larger than before (Ktor classes retained). No ProGuard warnings about Ktor classes.

**Done when**: Release APK builds successfully with Ktor keep rules applied.

</tasks>
