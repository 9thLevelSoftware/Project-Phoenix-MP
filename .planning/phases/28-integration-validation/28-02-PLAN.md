        pushCallCount++

        return pushResult

Test cases (~20):

**Auth state tests:**
- `sync with no token returns NotAuthenticated without calling push` — no token in settings → NotAuthenticated, fakeApi.pushCallCount == 0
- `login stores auth and returns user` — configure signInResult with valid GoTrueAuthResponse → verify tokenStorage.isAuthenticated becomes true
- `logout clears auth and sets NotAuthenticated` — setup authenticated, call logout → isAuthenticated false, syncState == NotAuthenticated

**Push success flow:**
- `sync pushes local changes and returns success` — setup auth, pushResult=success → syncState == Success
- `sync sends correct payload with deviceId and platform` — verify lastPushPayload.deviceId matches tokenStorage.getDeviceId(), platform is "android" or matches
- `push syncTime ISO 8601 parsed to epoch millis` — "2026-03-02T12:00:00Z" → syncState.syncTime matches Instant.parse("2026-03-02T12:00:00Z").toEpochMilliseconds()
- `sync with no local data sends empty payload` — sessionsToReturn=empty, routinesToReturn=empty → push still called with empty lists

**Pull success flow:**
- `sync merges routines from pull` — pullResult with 2 PullRoutineDtos → fakeSyncRepo.mergedRoutinesCaptured has size 2
- `sync merges badges from pull` — pullResult with 1 PullBadgeDto → fakeSyncRepo.mergedBadgesCaptured has size 1
- `sync merges gamification stats from pull` — pullResult with gamificationStats → fakeSyncRepo.mergedGamificationStatsCaptured != null
- `sync saves RPG attributes from pull` — pullResult with rpgAttributes → fakeGamificationRepo.savedRpgProfile != null
- `sync with empty pull response skips merge` — pullResult with empty lists → mergedRoutinesCaptured empty, mergedBadgesCaptured empty

**Error handling:**
- `push 401 sets NotAuthenticated state` — pushResult = failure(PortalApiException("Unauthorized", null, 401)) → syncState == NotAuthenticated
- `push non-401 error sets Error state with message` — pushResult = failure(PortalApiException("Server error", null, 500)) → syncState == Error("Server error")
- `pull failure is non-fatal and uses push syncTime` — pullResult = failure → syncState == Success, syncTime == push syncTime
- `push failure does not call pull` — pushResult = failure → pullCallCount == 0

**Timestamp management:**
- `sync updates lastSyncTimestamp in token storage` — after successful sync → tokenStorage.getLastSyncTimestamp() matches final syncTime
- `sync uses pull syncTime when larger than push` — push syncTime T1, pull syncTime T2>T1 → finalSyncTime == T2
- `sync uses push syncTime when pull fails` — pullResult = failure → finalSyncTime == push-derived time

**State transitions:**
- `initial state is Idle` — syncState.value == Idle before any sync call

## Step 5: Create PortalTokenStorageTest.kt

Create `shared/src/commonTest/kotlin/com/devil/phoenixproject/data/sync/PortalTokenStorageTest.kt`

Test cases (~7):
- `isTokenExpired returns true when token expired` — set expires_at to past → isTokenExpired() == true
- `isTokenExpired returns true within 60-second buffer` — set expires_at to (now + 30s) → isTokenExpired() == true (within 60s buffer)
- `isTokenExpired returns false when token has more than 60 seconds remaining` — set expires_at to (now + 120s) → isTokenExpired() == false
- `isTokenExpired returns true when no token stored` — fresh MapSettings → isTokenExpired() == true
- `clearAuth preserves deviceId and lastSyncTimestamp` — set all fields, call clearAuth() → deviceId and lastSync preserved, token gone
- `saveGoTrueAuth stores token and refresh token and user` — save valid response → getToken() returns access_token, getRefreshToken() returns refresh_token, currentUser has correct id/email
- `hasToken returns false after clearAuth` — save then clear → hasToken() == false

These tests validate that the proactive refresh trigger (isTokenExpired with 60s buffer) works correctly, which is the foundation of SC-3's "silent refresh" behavior.
